## Phase 3 - Incident Response

**Standard:** ISO 27001:2022  
**Document:** [View Playbook](playbook-phishing.md)

The third and final phase of this project involved building a phishing incident response playbook for Pay29 Solutions. The playbook was built around the ISO 27001:2022 Annex A incident 
management controls (A.5.24 to A.5.28) and walks through seven phases - Preparation, Detection, Analysis, Containment, Eradication, Recovery and Lessons Learned.

The Preparation phase defines what Pay29 should have in place before an incident occurs - things like a reporting channel, MFA enforcement and awareness of this playbook across relevant staff. 
Detection and Analysis cover how to identify a phishing incident and assess its scope before taking any action. 
Containment and Eradication focus on stopping the threat and removing it completely from the environment. 
Recovery deals with safely restoring affected accounts and verifying no data was exposed. 
The playbook closes with a Lessons Learned section to ensure every incident makes Pay29's security posture a little stronger.
