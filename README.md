# ISO 27001 Risk & Compliance Program for Cloud-Hosted Payment Platform

## Overview
A Governance, Risk, and Compliance (GRC) portfolio project built to demonstrate practical application of ISO 27001 in a fictional fintech environment.

## 🏢 About Pay29 Solutions
Pay29 is a fictional 120-person fintech company that processes online payments. They run on AWS, use Microsoft 365, and handle sensitive customer PII and transaction records, but have no formal security policies, no MFA, and no incident response plan. It is a growing startup that hasn't prioritized security yet. A phishing attack 3 months ago compromised 2 employee accounts. It was resolved manually with no documented process.
  
## Scope of Work
**Phase 1 - Gap Analysis**  
Conducted an ISO 27001:2022 gap analysis for Pay29 Solutions evaluating the organization against all seven mandatory clauses broken down into 25 sub-clauses. Documented compliance status, identified gaps and provided 
a prioritized set of immediate recommendations to address the most critical findings.

**Phase 2 - Risk Assessment**  
Built an asset inventory covering 10 key assets across Pay29's environment and conducted a full risk assessment using a 5x5 likelihood and impact matrix. Each risk was mapped to an ISO 27001:2022 Annex A control with 
a defined treatment action and risk owner.

**Phase 3 - Incident Response**  
Developed a phishing incident response playbook referencing ISO 27001 Annex A incident management controls (A.5.24 to A.5.28), directly informed by the phishing incident Pay29 experienced 3 months prior that exposed two employee accounts.
