## Phase 1 - Gap Analysis

**Standard:** ISO 27001:2022  
**Document:** [View Full Report](gap-analysis-report.pdf)

The first phase of this project involved conducting an ISO 27001:2022 gap analysis for Pay29 Solutions to understand where the organization currently stands before any security controls are put in place.
To do this I reviewed Pay29's existing security posture against all seven mandatory clauses of ISO 27001:2022 - breaking each clause down to the sub-clause level to get a more granular view. This gave a total of 25 sub-clauses assessed across areas like leadership, planning, support, operations and performance evaluation.

For each sub-clause I documented the current compliance status (Implemented, Partially Implemented or Not Implemented), identified the specific gap and provided a recommendation to address it.
The findings were not surprising for a company at Pay29's stage - 21 out of 25 sub-clauses were not implemented and 4 were only partially in place. The biggest gaps were around the absence of a formal security policy, no defined security roles, no risk assessment process and no incident response plan despite the organization having experienced a phishing attack three months prior.

The phase wraps up with a set of immediate priority recommendations covering the most critical gaps - including developing an information security policy, defining the ISMS scope, assigning security roles and enforcing MFA across all Microsoft 365 accounts.
